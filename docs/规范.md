# 校园二手交易平台数据库系统 — 项目规范

## 一、概述

本文档定义了 SecondHandSQL 项目的开发规范，涵盖代码风格、数据库设计、命名约定、文件组织等方面，确保项目保持一致性和可维护性。

---

## 二、项目结构规范

### 2.1 目录结构

```
secondhandsql/
├── app.py                  # Flask 主程序（入口）
├── schema.sql              # 数据库结构（表、约束、触发器、视图）
├── init_data.sql           # 初始数据
├── requirements.txt        # Python 依赖
├── run.bat                 # Windows 启动脚本
├── launcher.ps1            # 防火墙配置与启动器
├── build_portable.bat      # 打包脚本
├── secondhandsql.spec      # PyInstaller 配置
├── static/
│   └── style.css           # 全局样式
├── templates/
│   ├── base.html           # 基础模板
│   ├── index.html          # 首页
│   ├── users.html          # 用户列表
│   ├── items.html          # 商品列表
│   ├── orders.html         # 订单列表
│   └── queries.html        # 查询结果
├── dist/secondhandsql/     # 打包输出目录
└── docs/                   # 项目文档
```

### 2.2 文件组织原则

- **源码与资源分离**：Python 代码放在根目录，模板和静态资源分别放在 `templates/` 和 `static/`。
- **数据库文件不提交**：`secondhandsql.db` 为运行时生成，不纳入版本控制。
- **打包产物不提交**：`build/` 和 `dist/` 为构建输出，不纳入版本控制。

| 类型 | 规范 | 示例 |

| 模块/包 | 全小写，可用下划线 | `app.py`, `data_utils.py` |

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 全小写，单数名词 | `user`, `item`, `orders` |

## 四、代码风格规范

### 4.1 Python

- **遵循 PEP 8**，使用 4 空格缩进。
- **行长度**：建议不超过 100 字符。
- **导入分组**：标准库 → 第三方库 → 本地模块，每组之间空一行。
- **类型注解**：函数参数和返回值尽量添加类型注解。

```python
# 示例
from datetime import date
from pathlib import Path

from flask import Flask, render_template

def get_items_with_seller() -> list[sqlite3.Row]:
    return fetch_all("""
        SELECT i.item_id, i.item_name, u.user_name AS seller_name
        FROM item AS i
        JOIN user AS u ON u.user_id = i.seller_id
        ORDER BY i.item_id
    """)
```

### 4.2 SQL

- **关键字大写**：`SELECT`, `FROM`, `WHERE`, `JOIN`, `CREATE TABLE` 等。
- **标识符小写**：表名、列名使用小写。
- **格式化**：多行 SQL 使用缩进对齐，子句换行。

```sql
-- 示例
SELECT
    i.item_id,
    i.item_name,
    u.user_name AS buyer_name
FROM orders AS o
JOIN item AS i ON i.item_id = o.item_id
JOIN user AS u ON u.user_id = o.buyer_id
WHERE i.status = 1
ORDER BY o.order_date;
```

### 4.3 CSS

- **变量优先**：颜色、间距等使用 CSS 变量定义。
- **BEM 风格**：块 + 元素 + 修饰符的命名思路（本项目使用简化版 kebab-case）。
- **响应式**：使用 `@media` 处理移动端适配。

```css
/* 示例 */
.card {
    background: var(--surface);
    border: 1px solid var(--line);
    border-radius: 20px;
}

.button-danger {
    background: var(--danger);
}
```

## 五、数据库设计规范

### 5.1 表设计原则

- **每张表必须有主键**，优先使用业务主键（如 `user_id`）。
- **外键必须显式声明**，并指定 `ON UPDATE CASCADE` 和 `ON DELETE RESTRICT`。
- **CHECK 约束**：状态字段、日期格式等使用 CHECK 约束保证数据合法性。

### 5.2 字段类型约定

| 数据类型 | 使用场景 |
|----------|----------|
| `TEXT` | 字符串、日期（ISO 8601 格式） |
| `REAL` | 价格、浮点数 |
| `INTEGER` | 状态码、计数 |

### 5.3 一致性保证

- **触发器**：用于跨表一致性检查（如订单插入时商品必须为已售状态）。
- **唯一约束**：`orders.item_id` 唯一，保证每个商品最多交易一次。
- **事务**：业务操作（如购买）必须使用事务包裹，失败时回滚。

## 六、Flask 应用规范

### 6.1 路由与视图函数

- **路由命名**：使用小写 + 下划线，与视图函数名一致。
- **请求方法**：POST 路由用于数据修改，GET 路由用于数据展示。
- **闪存消息**：操作成功后使用 `flash()` 提供用户反馈。

```python
@app.route("/items")
def items():
    ...

@app.post("/items/add")
def add_item():
    ...
```

### 6.2 数据库连接

- **使用 `g` 对象**：每个请求周期内复用数据库连接。
- **自动关闭**：通过 `@app.teardown_appcontext` 确保连接释放。
- **外键启用**：每次连接后执行 `PRAGMA foreign_keys = ON`。

### 6.3 错误处理

- **捕获特定异常**：`sqlite3.IntegrityError`, `ValueError` 等。
- **用户友好提示**：错误信息使用中文，避免暴露底层细节。

## 七、模板规范

### 7.1 Jinja2 模板

- **继承基础模板**：所有页面继承 `base.html`。
- **块命名**：`title`, `content` 为标准块名。
- **过滤器使用**：`url_for()` 生成链接，避免硬编码 URL。

```html
{% extends "base.html" %}

{% block title %}商品列表{% endblock %}

{% block content %}
  <h2>商品列表</h2>
  ...
{% endblock %}
```

### 7.2 HTML 结构

- **语义化标签**：使用 `<header>`, `<main>`, `<section>`, `<nav>` 等。
- **无障碍属性**：添加 `lang="zh-CN"`, `charset="utf-8"` 等。

## 八、打包与交付规范

### 8.1 打包流程

1. 更新 `requirements.txt`。
2. 运行 `build_portable.bat` 生成打包目录。
3. 验证 `dist/secondhandsql/` 包含：
   - `secondhandsql.exe`
   - `run.bat`
   - `launcher.ps1`
   - `_internal/`（依赖库）

### 8.2 启动器规范

- `run.bat` 为统一入口，自动检测打包版或源码版。
- `launcher.ps1` 负责：
  1. 申请管理员权限（首次运行）。
  2. 清理防火墙阻止规则。
  3. 添加允许规则（程序级 + 端口级）。
  4. 启动应用。

## 十一、安全规范

### 11.1 应用层安全

- **SQL 注入防护**：所有数据库操作使用参数化查询，禁止字符串拼接 SQL。
- **XSS 防护**：模板中自动转义变量，不信任用户输入。
- **CSRF 防护**：本项目为演示用途，未启用 CSRF Token（生产环境建议启用）。

### 11.2 数据安全

- **敏感信息**：`SECRET_KEY` 等配置在演示环境中使用固定值，生产环境必须更换。
- **数据库备份**：定期导出 `secondhandsql.db` 文件作为备份。
