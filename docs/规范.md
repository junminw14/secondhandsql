# 校园二手交易平台数据库系统 - 项目规范

## 概述

本文档定义 SecondHandSQL 项目的开发规范，覆盖代码风格、数据库设计、命名约定和文件组织。

## 项目结构

```text
secondhandsql/
├── app.py                  # Flask 主程序
├── schema.sql              # 数据库结构
├── init_data.sql           # 初始数据
├── requirements.txt        # Python 依赖
├── run.bat                 # Windows 启动脚本
├── launcher.ps1            # 本机打包版启动器
├── build_portable.bat      # 打包脚本
├── secondhandsql.spec      # PyInstaller 配置
├── static/
│   └── style.css           # 全局样式
├── templates/
│   ├── base.html           # 基础模板
│   ├── index.html          # 首页
│   ├── users.html          # 用户列表
│   ├── items.html          # 商品列表
│   ├── orders.html         # 订单列表
│   └── queries.html        # 查询结果
├── dist/secondhandsql/     # 打包输出目录
└── docs/                   # 项目文档
```

## 运行规范

- Vercel 部署作为公网访问入口。
- 本地打包版只用于本机演示或离线演示。
- 本地 Flask 服务只绑定 `127.0.0.1:5000`。
- 启动器不申请管理员权限，不修改 Windows 防火墙规则。

## 代码规范

- Python 代码遵循 PEP 8。
- SQL 使用清晰的表名和字段名。
- 模板中只保留当前页面需要的数据，不传递未使用变量。
- 删除功能时同步清理依赖、模板变量、样式和文档说明。

## 数据库规范

- 所有表必须有主键。
- 外键关系需要在 schema 中显式声明。
- 写操作必须通过应用层校验权限。
- 购买商品这类跨表写入必须使用事务。

## 前端规范

- 页面文案保持简洁，避免展示过时的部署或启动信息。
- 首页展示系统入口、统计数据和管理员操作。
- 重复样式优先放在 `static/style.css` 中维护。
