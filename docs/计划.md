# 校园二手交易平台数据库系统 - 项目计划

## 一、目标

项目必须同时满足：

1. 网站能通过浏览器访问
2. 提交给老师后无需额外安装 Python
3. 尽量做到新手双击即用，少走防火墙和局域网访问弯路

## 二、技术方案

### 1. 数据库

使用 `SQLite`：

- 轻量
- 单文件
- 便于与项目一起打包

### 2. Web 框架

使用 `Python + Flask`：

- 开发快
- 页面和表单处理简单
- 与 SQLite 结合自然

### 3. 打包方式

使用 `PyInstaller --onedir`：

- 把 Python 运行时和依赖打进交付目录
- 目标机器无需额外安装 Python
- 保留整个目录即可运行

### 4. 启动方式

使用 `run.bat + launcher.ps1 + secondhandsql.exe` 组合：

- `run.bat`：统一入口
- `launcher.ps1`：自动申请管理员权限，修复常见防火墙阻断
- `secondhandsql.exe`：实际启动网站

## 三、对“像厂商互联一样方便”的实现策略

完全复制厂商互联的体验并不现实，因为它们通常有自家双端 App、常驻服务和更深的系统集成。  
但本项目可以做到接近的“一遍过”流程：

1. 双击启动
2. 若需要则弹出管理员确认
3. 自动清理阻止规则
4. 自动添加允许规则
5. 自动启动网站
6. 自动打开浏览器
7. 控制台和首页都显示二维码

## 五、实现范围

### 基础页面

- 首页
- 用户列表页
- 商品列表页
- 订单列表页
- 查询结果页

### 数据操作

- 插入新商品
- 修改商品价格
- 删除未售商品
- 购买商品

### 查询

- 基本查询 4 项
- 连接查询 3 项
- 聚合与分组 4 项
- 视图 2 项

### 一致性与业务逻辑

- 主键、外键、非空、检查约束
- 触发器保证订单与商品状态一致
- 购买事务
- 已售商品不可重复购买

## 六、启动器增强计划

### 1. 自动检测并清理 Block 规则

启动器扫描与 `secondhandsql.exe` 相关的入站阻止规则，并自动删除。

### 2. 自动申请管理员权限并添加 Allow 规则

启动器在首次运行时自动申请管理员权限，并添加：

- 程序级允许规则
- TCP 5000 端口允许规则

### 3. 自动打开浏览器

网站启动后自动打开本机地址，避免用户自己复制。

### 4. 控制台显示二维码

便于手机直接扫码访问。

### 5. 文档改成新手一遍过

老师或同学只看文档即可完成启动，不需要自己排障。

## 七、打包与验证

### 打包步骤

1. 安装依赖和 `PyInstaller`
2. 打包 `secondhandsql.exe`
3. 复制 `run.bat`、`launcher.ps1`、文档到 `dist/secondhandsql/`
4. 验证打包版启动
5. 验证网页可访问

### 验收标准

- 网站能在本机打开
- 打包版无需系统 Python
- 双击 `run.bat` 能自动进入正确流程
- 管理员确认后可自动修复常见防火墙阻断
- 首页和控制台都能提供手机访问方式

## 八、结论

最终交付不再只是“能打包运行”，而是：

- 能打包
- 能自动修复最常见的局域网访问阻断
- 能把新手步骤压缩到最少

这更接近你要的“像厂商互联一样方便”的交付体验。
