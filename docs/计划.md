# 校园二手交易平台数据库系统 - 项目计划

## 目标

1. 网站可以通过浏览器访问。
2. 项目已经部署到 Vercel，公网访问优先使用 Vercel 地址。
3. 打包版保留为本机演示入口，目标机器无需额外安装 Python。

## 技术方案

### 数据库

本地演示使用 SQLite：

- 轻量
- 单文件
- 便于和项目一起打包

线上部署使用环境变量中的 PostgreSQL 连接信息。

### Web 框架

使用 `Python + Flask`：

- 开发快
- 页面和表单处理简单
- 便于同时支持 SQLite 和 PostgreSQL

### 打包方式

使用 `PyInstaller --onedir`：

- 将 Python 运行时和依赖打入交付目录
- 目标机器无需额外安装 Python
- 保留整个目录即可运行

### 启动方式

使用 `run.bat + launcher.ps1 + secondhandsql.exe`：

- `run.bat`：统一入口
- `launcher.ps1`：解析可执行文件路径并启动应用
- `secondhandsql.exe`：实际启动网站

本地启动后浏览器访问 `http://127.0.0.1:5000`。

## 验收标准

- Vercel 公网访问可用。
- 本地运行只显示本机访问方式。
- 首页不显示局域网 IP 或二维码。
- 启动脚本不申请管理员权限。
- 项目不依赖二维码生成库。
- Python 文件可以通过语法检查。
